# Markdownで画像を保存・使用する方法

## 📁 フォルダ構造

```
CURSOR-TUOR/
├── Season2ー第3回.md          # Markdown記事
├── images/                    # 画像フォルダ
│   ├── README.md
│   ├── image1.jpg
│   └── image2.png
└── ...
```

## 🖼️ 画像の保存手順

### 1. 画像ファイルを準備
- スマートフォンやカメラで撮影
- スクリーンショットを取得
- インターネットからダウンロード（著作権に注意）

### 2. 画像を `images` フォルダに保存
- ファイル名はわかりやすく（例: `qrcode-example.png`）
- 日本語のファイル名も使用可能ですが、英数字推奨

### 3. Markdownで画像を参照

#### 基本的な画像表示
```markdown
![画像の説明テキスト](images/画像ファイル名.jpg)
```

#### 実際の例
```markdown
![QRコードの例](images/qrcode-example.png)
![エスプレッソマシン](images/espresso-machine.jpg)
```

## 📝 画像の記法の詳細

### 基本構文
```markdown
![代替テキスト](画像のパス)
```

- **代替テキスト**: 画像が表示できない場合に表示されるテキスト
- **画像のパス**: 画像ファイルへの相対パスまたは絶対パス

### サイズ指定（HTML記法を使用）
```markdown
<img src="images/example.jpg" alt="説明" width="500">
```

### 画像にリンクを付ける
```markdown
[![画像の説明](images/example.jpg)](https://example.com)
```

## 🎯 実践例：記事に画像を追加

### 例1: QRコードの画像を追加
```markdown
店舗のマシンやバックヤードには、いくつもの「QRコード」が貼られていた。

![QRコードの例](images/qrcode-example.png)

「お、これっすか」
健太がスマホをかざす。
```

### 例2: 複数の画像を並べて表示
```markdown
### 清掃手順の比較

**正しい方法:**
![正しい清掃方法](images/cleaning-correct.jpg)

**間違った方法:**
![間違った清掃方法](images/cleaning-wrong.jpg)
```

## 💡 ベストプラクティス

### ✅ 推奨事項
1. **画像フォルダを整理**: すべての画像を `images` フォルダに保存
2. **わかりやすいファイル名**: `2024-01-15-coffee-machine.jpg` のように日付や内容を含める
3. **適切な画像サイズ**: 大きすぎる画像は読み込みが遅くなるため、必要に応じて圧縮
4. **代替テキストを記入**: アクセシビリティのため、必ず代替テキストを記入

### ❌ 避けるべきこと
1. 画像をMarkdownファイルと同じフォルダに直接保存（整理が大変）
2. 絶対パスを使用（他の環境で動かなくなる）
3. 代替テキストを空にする

## 🔄 画像の更新方法

1. 古い画像ファイルを `images` フォルダから削除
2. 新しい画像を同じファイル名で保存（参照を変更する必要なし）
3. または、新しいファイル名で保存し、Markdownの参照を更新

## 📱 ブラウザでの表示

HTMLに変換した際も、画像は正しく表示されます：
- 相対パスがそのまま機能します
- `contents-test-view.html` のようなHTMLファイルでも画像が表示されます

## 🛠️ 画像編集ツール（参考）

画像を編集する場合のツール例：
- **macOS**: プレビューアプリ（トリミング、サイズ変更）
- **オンライン**: Canva, Photopea
- **アプリ**: GIMP, Photoshop
